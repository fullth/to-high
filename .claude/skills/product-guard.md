# 프로덕트 가드 스킬

## 🎯 프로젝트 핵심 가치

### To High; 위로의 정체성
1. **공감과 위로 중심**
   - 해결보다는 경청과 지지
   - 비판단적 태도
   - 안전한 대화 공간

2. **단계적 깊이**
   - 처음엔 가벼운 질문
   - 점진적으로 깊이 파고들기
   - 사용자가 스스로 깨닫도록 유도

3. **접근성**
   - 24시간 언제든 이용
   - 익명성 보장 (비로그인 가능)
   - 낮은 진입장벽

4. **전문성**
   - 심리 상담 가이드라인 준수
   - 위기 감지 및 전문가 안내
   - 윤리적 책임

## ⚠️ 절대 변경 금지 사항

### UX 철학
- ❌ 사용자를 판단하는 표현
- ❌ 강압적인 조언이나 지시
- ❌ 너무 빠른 문제 해결 시도
- ❌ 개인정보 과도 요구
- ❌ 유료 기능의 강제

### 기술적 안전장치
- ❌ 위기 감지 시스템 우회
- ❌ 사용자 입력 검증 제거
- ❌ 세션 컨텍스트 무제한 (토큰 폭증)
- ❌ Rate Limiting 제거 (악용 가능)

### 데이터 보호
- ❌ 상담 내용 제3자 공유
- ❌ 동의 없는 데이터 수집
- ❌ 프로필 학습 강제

## 📋 변경사항 검수 체크리스트

### 1. UX 영향 분석
```
□ 사용자가 불편해지지 않는가?
□ 진입장벽이 높아지지 않는가?
□ 기존 사용자 흐름이 깨지지 않는가?
□ 익명성이 보장되는가?
□ 강제성이 없는가?
```

### 2. 핵심 가치 보존
```
□ 공감과 위로의 톤이 유지되는가?
□ 비판단적 태도가 유지되는가?
□ 안전한 대화 환경이 유지되는가?
□ 단계적 깊이 접근이 유지되는가?
```

### 3. 윤리적 검토
```
□ 위기 감지 시스템이 작동하는가?
□ 전문가 연계가 적절한가?
□ 데이터 보호가 충분한가?
□ 약관/동의가 명확한가?
```

### 4. 비즈니스 임팩트
```
□ 무료 사용자 가치를 해치지 않는가?
□ 레거시 사용자 혜택이 유지되는가?
□ 구독 가치가 명확한가?
□ 과도한 비용 발생은 없는가?
```

## 🚨 검수 실패 사례

### ❌ 나쁜 예시 1: 로그인 강제
```
기획: "상담 시작 전 무조건 로그인"
문제: 진입장벽 상승, 익명성 파괴
대안: 3회까지 비로그인 허용 (현재 방식 유지)
```

### ❌ 나쁜 예시 2: 즉시 조언 모드
```
기획: "첫 질문부터 해결책 제시"
문제: 공감 단계 생략, 사용자 부담 증가
대안: 3턴 이후 조언 모드 (현재 방식 유지)
```

### ❌ 나쁜 예시 3: 개인정보 수집
```
기획: "상담 전 나이/성별/직업 필수 입력"
문제: 진입장벽, 프라이버시 침해
대안: 선택적 프로필, 대화 중 자연스럽게 파악
```

### ❌ 나쁜 예시 4: 컨텍스트 무제한
```
기획: "대화 횟수 제한 제거"
문제: 토큰 비용 폭증, 품질 저하
대안: Rolling Summary 유지 (20턴마다)
```

## ✅ 검수 통과 사례

### ✅ 좋은 예시 1: 세션 재개
```
기획: "이전 상담 이어서 하기"
체크: ✓ 진입장벽 낮춤 ✓ 연속성 제공 ✓ 개인정보 과도수집 없음
승인: 사용자 편의성 향상
```

### ✅ 좋은 예시 2: 카테고리 선택
```
기획: "시작 시 고민 주제 선택"
체크: ✓ 선택적 ✓ 건너뛰기 가능 ✓ 진입장벽 낮음
승인: 맞춤형 대화 시작
```

### ✅ 좋은 예시 3: 위기 감지 강화
```
기획: "자해/자살 언급 시 즉시 전문기관 안내"
체크: ✓ 윤리적 책임 ✓ 안전장치 강화 ✓ 강제 아님
승인: 핵심 가치 보강
```

## 🔍 검수 프로세스

### Step 1: 영향 범위 파악
```
1. 어떤 기능을 변경/추가하는가?
2. 사용자 여정의 어느 단계인가?
3. 필수인가, 선택인가?
4. 기존 기능과의 관계는?
```

### Step 2: 체크리스트 실행
```
위 4가지 검수 체크리스트 모두 통과해야 함
하나라도 실패 시 → 재기획 or 대안 제시
```

### Step 3: 리스크 평가
```
낮음: 부분 UI 개선, 버그 수정
중간: 새 기능 추가, 흐름 변경
높음: 핵심 로직 변경, 정책 변경
→ 높음일수록 신중한 검토
```

### Step 4: 승인/거부
```
✅ 승인: 모든 체크리스트 통과
⚠️  조건부 승인: 일부 개선 필요
❌ 거부: 핵심 가치 위배, 재기획 필요
```

## 💡 사용 방법

2. 검수 템플릿 작성
3. 리스크가 높으면 더 신중히 검토
