# Safety Review

심리 상담 서비스의 특수성을 반영한 감사 체크리스트.

## 감사 4축

### 1. 위기 감지 무결성 (최우선)

**파일**: `apps/api/src/common/crisis-detector.ts`, `crisis-detector.spec.ts`

- [ ] 키워드 제거·레벨 다운그레이드가 있다면 근거 명시? 테스트 통과?
- [ ] `level: 'high'`가 탐지될 때 상담사 응답보다 먼저 위기 안내가 반환되는가?
- [ ] 프롬프트 변경이 위기 신호를 덮지 않는가 (예: "긍정적으로 생각해보세요"로 위기 감정 회피하는 응답 유도)
- [ ] 위기 감지된 사용자의 context가 일반 rolling summary에 삼켜져 기록 소실되지 않는가?
- [ ] UI(`apps/web`)에서 위기 응답이 올 때 전문기관 안내 배너가 애니메이션 없이 즉시 노출되는가?
- [ ] 자살예방상담전화 `1393` 번호가 잘못 수정되지 않았는가?

### 2. PII 보호

- [ ] 세션 context(사용자 원문)가 Sentry/일반 로그에 평문으로 들어가지 않는가?
- [ ] OpenAI로 전달되는 context는 합의된 범위 내인가?
- [ ] 사용자 프로필(이메일, OAuth provider ID)이 응답 DTO에 필요 이상으로 포함되지 않는가?
- [ ] Admin 엔드포인트가 사용자 context를 반환한다면 접근 권한이 엄격한가?
- [ ] 에러 응답이 내부 예외 메시지(`ValidationError: ...`)를 사용자에게 노출하지 않는가?

### 3. 인증·권한·구독 우회

- [ ] `userId === 'anonymous'` 분기에서 권한 있는 리소스 접근이 막혀 있는가?
- [ ] `JwtGuard` vs `OptionalJwtGuard` 선택이 엔드포인트 의도와 일치하는가?
- [ ] 세션 제한(`FREE_USER_SESSION_LIMIT`, 구독 티어)이 클라이언트에서 우회 가능한가?
- [ ] 구독 만료 시 세션 생성이 차단되는가?
- [ ] Admin API에 권한 체크가 모든 경로에 적용되는가?
- [ ] OAuth 콜백에서 토큰이 URL/로그에 노출되지 않는가?

### 4. 상담 윤리

- [ ] 응답 프롬프트가 의료 진단·약물 조언을 유도하지 않는가?
- [ ] 판단적 언어("그건 잘못된 생각이에요")를 강화하지 않는가?
- [ ] 사용자 감정을 최소화하는 표현("별거 아니에요")이 생성될 여지가 있는가?
- [ ] 아동·청소년 관련 내용에 별도 가드가 있는가?
- [ ] 개인 식별 가능한 타인(가족, 직장 상사 실명)을 UI나 응답에 그대로 출력하지 않는가?

## 감사 절차

1. **변경 파일 목록 수집** — api-architect/web-architect/prompt-engineer가 제공한 파일 diff 분석
2. **4축 체크리스트 적용** — 해당 축만 체크(전체 수행 불필요)
3. **이슈 분류**:
   - **Critical** — 사용자에게 즉시 해를 끼칠 수 있음 (위기 감지 약화, PII 평문 로그, 위기 UI 누락)
   - **High** — 릴리즈 전 수정 필요 (권한 검증 누락, 판단적 표현 유도)
   - **Medium** — 개선 권장 (오탐 증가 가능성, 에러 메시지 톤)
   - **Low** — 참고
4. **보고서 작성** — `_workspace/NN_safety-auditor_audit.md`
5. **수정 위임** — 직접 패치 금지. 소유 에이전트에 요청.
